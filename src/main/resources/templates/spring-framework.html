Spring Framework

Spring ApplicationApril 2020

Hierbei handelt es sich um die Webseite einer imaginären Tierklinik.

Es können die Besitzer mit dem jeweiligen Haustier durch ein Suchfeld aus der Datenbank herausgesucht werden. Darüber hinaus können Ärzte mit ihrem jeweiligen Spezialgebieten angezeigt werden. Neue Besitzer und Haustiere können hinzugefügt werden, werden aber auf Grund der verwendeten H2 Datenbank nicht persistiert.

Um Daten zu persistieren muss eine Datenbank, zum Beispiel MySQL, im pom.xml als dependency eingebunden und in den application.properties konfiguriert werden.

Hier gehts zum GitHub Repository.

Erläuterung zum Programmverlauf

Wie im Video gezeigt können den Haustieren, der jeweiligen Besitzer neue Besuche hinzugefügt oder die Daten der Besitzer verändert bzw. ein neues Haustier hinzugefügt werden. Darüber hinaus kann auch ein neuer Besitzer mit Haustier hinzugfügt werden.

Dafür leitet der jeweilige Controller, über RequestMapping zur gewünschten Seite weiter. In der Serviceklasse ist die Logik gesammelt und führt den jeweiligen Befehl aus. Über das CrudRepository stehen die Methoden zum erstellen, auslesen, updaten und löschen von Daten zur verfügung. Eigene Methoden können aber jederzeit hinzugefügt werden. Durch PostMapping im jeweiligen Controller werden die Daten hinzugefügt.

Die Daten werden aktuell noch nicht persistiert, sondern nur in der H2 Datenbank gespeichert. Eine MySQL Datenbank werde ich im späteren Verlauf noch hinzufügen, die dependency ist schon im pom.

Das triggern einer Exception ist über den Errorbutton zu erreichen. Es wird zu einer neuen Page weitergeleitet, dort wird die die Message der Runtime Exception angezeigt.

Um die Liste der Ärzte anzeigen zu können wurde eine foreach-loop von Thymeleaf verwendet. Thymeleaf ist eine template engine für HTML, ein Browser versteht also Thymleaf als HTML. In der loop werden die einzelnen Ärzte mit ihren Spezialgebieten ausgegeben. Dazu gibt es ein Bild im Carousel über die Spring Pet Clinic.
